# Development

## Running from source

```bash
git clone https://github.com/keepur/beekeeper.git
cd beekeeper
npm ci
npm run build
```

That produces `dist/cli.js` and `dist/index.js`. Either:

- `npm link` in the checkout to put `beekeeper` + `beekeeperd` on your PATH (recommended), or
- run via `node dist/cli.js <subcommand>` directly.

For quick iteration without a build, `npm run dev` runs the daemon under `tsx`.

## Tests

```bash
npm run check      # what CI runs: typecheck + test
npm run test       # vitest run
npm run test:watch # vitest in watch mode
```

Tests live beside the source they cover (`src/foo.ts` ↔ `src/foo.test.ts`). Node 22+ required.

There is no lint or format step — don't add one without discussion.

## CI

CI runs on a self-hosted macOS ARM64 runner (`.github/workflows/ci.yml`). Don't switch to a GitHub-hosted runner without discussion — the runner hosts the same stack the production deploy uses.

## Branch protection

`main` is protected with `enforce_admins: true`: every change goes through a PR, no exceptions. PRs must pass the `Typecheck + Test + Build` check before merging. No force-pushes, no deletions, linear history only.

## Releases

Releases are tag-triggered (`.github/workflows/publish.yml`). The flow:

1. **Bump on a branch:**
   ```bash
   git checkout -b release/vX.Y.Z
   npm version --no-git-tag-version <patch|minor|major>
   git add package.json package-lock.json
   git commit -m "X.Y.Z"
   git push -u origin release/vX.Y.Z
   ```

2. **Open a PR**, wait for CI, merge.

3. **Tag the merged commit:**
   ```bash
   git checkout main
   git pull --ff-only origin main
   git tag vX.Y.Z
   git push origin vX.Y.Z
   ```

The publish workflow verifies that `package.json`'s version matches the tag, runs the full pipeline, then publishes `@keepur/beekeeper` to npm. A mistyped tag fails fast instead of publishing a wrong version.

npm auth is picked up from the self-hosted runner's ambient `~/.npmrc`. If you ever move the runner, add an `NPM_TOKEN` secret and write it to a workspace `.npmrc` before the `Publish` step.

## Gotchas

- **Claude Agent SDK `exports` regression** (≥ 0.2.101): the SDK bundles its CLI at `./cli.js` but doesn't list that subpath in `exports`. `session-manager.ts` resolves the path manually — don't remove that workaround until the SDK fixes its `exports`.
- **launchd minimal PATH**: `gui/<uid>` LaunchAgents inherit a minimal PATH (`/usr/bin:/bin:/usr/sbin:/sbin`). The wrapper script generated by `beekeeper install` exports a sane PATH BEFORE sourcing the env file so `node`, `git`, `npm`, etc. are reachable.
- **Team-proxy is opaque on purpose.** `team-proxy.ts` forwards raw frames. Don't add logging that assumes frame contents (would require parsing arbitrary third-party payloads).
